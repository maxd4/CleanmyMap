# UI Inventory Contract

## Purpose

The UI inventory baseline captures intended Streamlit UI wiring so drift can be reviewed explicitly.

## Inventory schema

The baseline generated by `scripts.ui_inventory` includes at least:
- `generated_at`
- `tabs` (tab id, label key, target renderer/module)
- `renderers` (module, function)
- `admin_components` (auth, map_review, moderation, exports references)
- `unreferenced_renderers`
- `missing_references`

Compatibility keys are also present to keep `ci_cleanup.py` checks stable.

## When to regenerate

Run `python -m scripts.ui_inventory regenerate --write-baseline` only when UI structure changes are intentional, reviewed, and expected.

Do not regenerate baseline to hide accidental drift.

## Who should run it

Developers/maintainers responsible for UI architecture and CI maintenance.

## How it differs from cleanup check

- Inventory `check` focuses on baseline drift detection.
- Cleanup diagnostic (`python -m scripts.ui_inventory cleanup --dry-run`) focuses on stale/orphan references and hygiene guidance.
- CI cleanup check (`python scripts/ci_cleanup.py --root . --check`) remains an explicit hard gate in primary CI.
