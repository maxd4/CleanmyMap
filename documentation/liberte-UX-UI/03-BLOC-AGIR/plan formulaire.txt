Objectif :
Rendre cette page finale plus claire, plus rassurante, plus professionnelle et plus utile avant validation, sans modifier la logique métier existante.

À améliorer en priorité :

1. Hiérarchie visuelle
- Renforcer la lisibilité du titre “Vérification et envoi”.
- Rendre le sous-texte plus direct : l’utilisateur doit comprendre qu’il vérifie avant publication officielle.
- Clarifier la structure globale :
  - score de fiabilité ;
  - résumé des données ;
  - points d’attention ;
  - action finale.
- Mieux hiérarchiser les informations importantes avant le bouton de publication.

2. Carte “Analyse de fiabilité”
- Clarifier le score affiché : actuellement “30 niveau” n’est pas assez explicite.
- Remplacer ou compléter par un libellé clair : “Score de fiabilité : 30/100” ou équivalent selon la logique existante.
- Expliquer brièvement pourquoi le score est faible/moyen/élevé.
- Ajouter une lecture qualitative : faible, moyen, bon, excellent, si cela existe ou peut être déduit proprement.
- Vérifier que le texte “Votre déclaration est prête à être transmise” n’est pas contradictoire avec les points d’attention.
- Si des informations manquent, préférer un message du type : “Déclaration transmissible, mais certains éléments peuvent être améliorés.”
- Améliorer le contraste, l’espacement, les alignements et la lisibilité de la carte sombre.

3. Cartes de résumé
- Harmoniser les cartes “La récolte”, “Le tracé”, “Le contexte”.
- Améliorer les titres, icônes, séparateurs, espacements et valeurs.
- Rendre les unités plus cohérentes : kg, unités, personne(s).
- Corriger les incohérences de casse : “25.5 KG” devrait être plus lisible comme “25,5 kg” si le projet utilise le format français.
- Corriger les libellés : “1 PERS.” peut devenir “1 personne”.
- Vérifier que les valeurs importantes ressortent sans surcharger.
- Ajouter un état vide clair pour les informations absentes.

4. Carte “Le tracé”
- Le message actuel indique qu’aucun tracé manuel n’est retenu et qu’un point GPS approximatif sera utilisé.
- Rendre ce message plus visible et plus compréhensible.
- Distinguer clairement :
  - tracé manuel fourni ;
  - géolocalisation approximative ;
  - aucun tracé disponible.
- Ajouter si possible un résumé : départ, arrivée, distance estimée, mode de tracé, précision.
- Si aucun tracé précis n’est fourni, proposer une action visible : “Retourner compléter le parcours”.
- Ne pas bloquer l’envoi si la logique actuelle autorise l’envoi sans tracé, mais avertir clairement.

5. Carte “Le contexte”
- Rendre les informations plus lisibles : date, structure, lieu.
- Corriger les valeurs incomplètes ou ambiguës, par exemple “Entreprise -” si cela vient d’une donnée manquante.
- Ajouter des placeholders propres : “Non renseigné” plutôt qu’un tiret ambigu.
- Vérifier que les données de l’étape 1 remontent correctement.
- Ajouter éventuellement un lien/bouton discret pour revenir modifier cette section.

6. Points d’attention
- Améliorer fortement le bloc “Points d’attention suggérés”.
- Rendre chaque point d’attention actionnable :
  - expliquer le problème ;
  - dire pourquoi c’est important ;
  - proposer une correction ou un retour vers l’étape concernée.
- Exemple : si le poids déclaré paraît incohérent avec le nombre de bénévoles et la durée, afficher un message plus précis et un bouton “Modifier les effectifs” ou “Modifier la collecte”.
- Hiérarchiser les alertes par gravité : critique, attention, conseil.
- Ne pas afficher un bloc d’alerte trop vague ou anxiogène.
- Vérifier contraste, couleur, icône, padding et lisibilité.

7. Bouton final “Publier ma déclaration”
- Clarifier que c’est l’action finale et engageante.
- Ajouter un état loading pendant l’envoi.
- Ajouter un état disabled si les conditions minimales ne sont pas remplies.
- Ajouter une confirmation de succès après publication.
- Ajouter une gestion d’erreur claire si l’envoi échoue.
- Vérifier que le bouton est suffisamment visible, mais pas ambigu.
- Sur mobile, s’assurer qu’il ne masque pas le contenu.

8. Navigation et correction avant envoi
- Ajouter ou vérifier la possibilité de revenir facilement aux étapes précédentes.
- Ajouter des actions “Modifier” sur les cartes importantes si cela respecte les patterns existants.
- L’utilisateur doit pouvoir corriger la récolte, le tracé ou le contexte sans perdre ses données.
- Vérifier que le retour conserve les informations déjà saisies.

9. UX globale
- L’utilisateur doit comprendre en moins de 5 secondes :
  - que sa déclaration est presque prête ;
  - quelles informations vont être envoyées ;
