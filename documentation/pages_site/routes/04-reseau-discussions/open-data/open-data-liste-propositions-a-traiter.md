# Données publiques - Liste des propositions à traiter

## Données

- Clarifier les formats exportables et leur usage
- Simplifier la présentation des accès API et des jeux de données
- Vérifier que les libellés restent courts et compréhensibles

## Lisibilité

- Réduire la densité des cartes techniques
- Donner une entrée plus directe vers les actions d'export
- Éviter la surcharge d'explications sur la couche de transparence

## Navigation

- Garder la page distincte de `Communauté` et de `Messagerie`
- Ne pas introduire de raccourci vers un annuaire ou un onboarding
- Maintenir une couleur dominante cohérente avec le bloc 04

## Alignement produit

- La page doit rester un point d'accès aux données, pas une page de support
- Les contenus techniques doivent rester lisibles par un public non spécialiste
