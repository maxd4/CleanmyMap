# Audit visual-first global

Date: 2026-05-19

## Objet

Ce document suit la couverture visuelle des documents qui doivent etre compris rapidement.
L'objectif n'est pas de faire du visuel partout, mais d'en mettre la ou il reduit vraiment la charge de lecture.

## Etat general

- Architecture et onboarding technique : couverture plutot bonne ;
- Produit : couverture heterogene, avec des pages encore trop textuelles ;
- Donnees et methode : besoin de schemas plus explicites ;
- Securite et maintenance : besoin de parcours visuels plus nets ;
- Pages de lecture rapide : bonnes candidates pour des flowcharts, des matrices et des tables courtes.

## Regle de priorisation

Quand un document depasse un simple rappel de contexte, il doit preferer :

1. un flowchart ;
2. une matrice courte ;
3. un tableau de synthese ;
4. un bloc de texte seulement si le visuel n'apporte rien.

## Ce qu'on doit surveiller

- les docs qui racontent le meme sujet sous plusieurs formes ;
- les pages longues sans ancrage visuel ;
- les fichiers qui gardent un fallback image mort ;
- les documents dont la structure a change mais pas les liens.

## Usage

Ce fichier sert de snapshot de travail. Quand une page produit est simplifiee ou restructuree, cette note doit etre re-lue et ajustee.
