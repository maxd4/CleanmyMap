# Audit Fast Origin Transfer de CleanMyMap

Derniere revue: 2026-06-20

Ce document complete [vercel-route-cost-audit.md](./vercel-route-cost-audit.md) et
[vercel-surface-report.md](./vercel-surface-report.md).
Objectif: identifier les causes les plus probables d'une consommation Vercel
`Fast Origin Transfer` anormalement elevee sur un site peu frequente.

Lecture:
- le rang indique la culpabilite probable pour `Origin Transfer`, du plus suspect au moins suspect;
- `Probabilite` = niveau de confiance dans le diagnostic, pas severite globale;
- les recommendations restent ciblees, sans refactor global.
- `État réel du repo` sert de colonne de suivi pour reprendre le backlog a partir de septembre 2026.

## Lecture utile

- Le suspect principal pour `Fast Origin Transfer` n'est pas le middleware.
- Les deux familles les plus probables sont les gros JSON appelés en `no-store` et les chunks Leaflet charges trop tot.
- La cartographie n'est pas forcement un proxy Vercel dans ce depot: `fetchMapActions()` passe deja par Supabase direct. Le cout restant est surtout le poids du chunk client.
- Les documents PDF/CSV/JSON generes cote serveur sont plus suspects que les images Supabase Storage dans l'etat actuel du code.

## Quick Wins Deja Traites

- `apps/web/src/components/reports/web-document/use-reports-web-document-model.ts`: suppression d'une requete SWR dupliquee pour les actions approuvees.
- `apps/web/src/components/sections/rubriques/use-annuaire-logic.ts` et `apps/web/src/app/api/partners/published-directory/route.ts`: lecture de l'annuaire public avec cache HTTP court au lieu de `no-store`.
- `apps/web/src/components/dashboard/funnel-conversion-panel.tsx` et `apps/web/src/app/api/gamification/analytics/funnel/route.ts`: cache court sur l'agrégat funnel public.
- `apps/web/src/components/sections/rubriques/use-kit-data.ts` et `apps/web/src/components/sections/rubriques/guide-section.tsx`: suppression des synchronisations serveur quand l'utilisateur n'est pas connecté.

## Priorite d'analyse pour la suite

1. Mesurer les hits reels de `/api/actions` et des routes `reports/*`.
2. Verifier quelles pages montent encore Leaflet avant visibilite.
3. Confirmer quels panels appellent plusieurs `no-store` au chargement initial.
4. Separer les compteurs simples des tableaux detailes.
