✅ `new URL()` + `protocol === \"https:\"`\n\n2. **Injection HTML**\n   - ❌ `innerHTML = userInput`\n   - ❌ `dangerouslySetInnerHTML` sans sanitisation\n   - ✅ `textContent = userInput`\n   - ✅ `{content}` en React\n   - ✅ `DOMPurify.sanitize()` si HTML nécessaire\n\n### Checklist avant commit\n\n- [ ] Pas de `startsWith(\"http\")` pour valider des URLs\n- [ ] Pas de `innerHTML` avec contenu utilisateur\n- [ ] Pas de `dangerouslySetInnerHTML` sans sanitisation\n- [ ] Utiliser `textContent` pour du texte simple\n- [ ] Utiliser `new URL()` pour parser les URLs\n\n---\n\n## 🚀 Comment utiliser cette documentation\n\n### Pour un développeur\n\n1. **Première fois:** Lire [SECURITY_QUICK_REFERENCE.md](./security/SECURITY_QUICK_REFERENCE.md) (5 min)\n2. **Pendant le développement:** Consulter [DEVELOPER_WORKFLOW.md](./security/DEVELOPER_WORKFLOW.md)\n3. **Avant de committer:** Utiliser la checklist\n4. **Besoin d'exemples:** Consulter [PATTERNS_VISUAL_GUIDE.md](./security/PATTERNS_VISUAL_GUIDE.md)\n\n### Pour un reviewer\n\n1. **Avant la revue:** Consulter [CODE_REVIEW_CHECKLIST.md](./security/CODE_REVIEW_CHECKLIST.md)\n2. **Pendant la revue:** Utiliser la checklist\n3. **Besoin de référence:** Consulter [SECURITY_QUICK_REFERENCE.md](./security/SECURITY_QUICK_REFERENCE.md)\n\n### Pour trouver rapidement\n\n→ Consulter [INDEX.md](./security/INDEX.md) pour la navigation\n\n---\n\n## 📈 Bénéfices\n\n### Pour les développeurs\n- ✅ Guide rapide et accessible\n- ✅ Exemples concrets\n- ✅ Checklist pour les revues\n- ✅ Commandes de vérification\n\n### Pour la sécurité\n- ✅ Prévention des vulnérabilités CodeQL courantes\n- ✅ Documentation centralisée\n- ✅ Traçabilité des corrections\n- ✅ Références aux standards OWASP/CWE\n\n### Pour la maintenabilité\n- ✅ Réduction des erreurs\n- ✅ Onboarding plus rapide\n- ✅ Documentation liée aux fichiers\n- ✅ Historique des corrections\n\n---\n\n## 🔗 Accès rapide\n\n### Guides essentiels\n- [SECURITY_QUICK_REFERENCE.md](./security/SECURITY_QUICK_REFERENCE.md) -
