# Intégration de la Documentation Sécurité dans le Workflow\n\nGuide pour intégrer les bonnes pratiques de sécurité dans votre workflow de développement.\n\n---\n\n## 🚀 Avant de commencer à coder\n\n### 1. Lire le guide rapide (5 min)\n```bash\n# Ouvrir le guide rapide\ncat documentation/security/SECURITY_QUICK_REFERENCE.md\n```\n\n**À retenir :**\n- ❌ Pas de `startsWith(\"http\")` pour valider les URLs\n- ❌ Pas de `innerHTML` avec contenu utilisateur\n- ✅ Utiliser `hasHttpsProtocol()` pour les URLs\n- ✅ Utiliser `textContent` pour du texte simple\n\n---\n\n## 💻 Pendant le développement\n\n### 1. Validation d'URL\n\n**Besoin :** Vérifier qu'une URL est en HTTPS\n\n```typescript\n// ✅ Copier-coller cette fonction\nfunction hasHttpsProtocol(url: string | undefined): boolean {\n  if (!url || typeof url !== \"string\") return false;\n  try {\n    const parsed = new URL(url);\n    return parsed.protocol === \"https:\";\n  } catch {\n    return false;\n  }\n}\n\n// Utilisation\nif (hasHttpsProtocol(userInput)) {\n  // Safe to use URL\n}\n```\n\n**Besoin :** Vérifier le hostname\n\n```typescript\n// ✅ Copier-coller cette fonction\nfunction isAllowedDomain(url: string, allowedHostname: string): boolean {\n  try {\n    const parsed = new URL(url);\n    return parsed.hostname === allowedHostname;\n  } catch {\n    return false;\n  }\n}\n\n// Utilisation\nif (isAllowedDomain(workshop.sourceUrl, \"www.academieduclimat.paris\")) {\n  // Safe\n}\n```\n\n### 2. Injection HTML\n\n**Besoin :** Afficher du texte simple\n\n```typescript\n// ✅ Utiliser textContent\nelement.textContent = \"↩\";\n\n// ❌ NE PAS utiliser innerHTML\nelement.innerHTML = \"↩\"; // DANGEREUX\n```\n\n**Besoin :** Afficher du contenu React\n\n```typescript\n// ✅ Laisser React sanitiser\n<div>{translationText}</div>\n\n// ❌ NE PAS utiliser dangerouslySetInnerHTML\n<div dangerouslySetInnerHTML={{ __html: t(\"footer.partner\") }} />\n```\n\n**Besoin :** Afficher du HTML contrôlé\n\n```typescript\n// ✅ Sanitiser avec DOMPurify\nimport DOMPurify from \"dompurify\";\n\n<div \n  dangerouslySetInnerHTML={{ \n    __html: DOMPurify.sanitize(dirtyHtml) \n  }} \n/>\n```\n\n**Besoin :** Ajouter des styles CSS\n\n```typescript\n// ✅ Utiliser style JSX\n<style jsx>{`\n  @media print {\n    body { -webkit-print-color-adjust: exact; }\n  }\n`}</style>\n\n// ✅ Ou utiliser la prop style\n<div style={{ color: \"red\" }}>Texte</div>\n\n// ❌ NE PAS utiliser dangerouslySetInnerHTML pour les styles\n<style dangerouslySetInnerHTML={{ __html: printStyles }} />\n```\n\n---\n\n## 🔍 Avant de committer\n\n### 1. Vérifier les patterns dangereux\n\n```bash\n# Chercher les startsWith(\"http\")\ngrep -r \"startsWith.*http\" apps/web/src --include=\"*.ts\" --include=\"*.tsx\"\n\n# Chercher les innerHTML\ngrep -r \"\\.innerHTML\" apps/web/src --include=\"*.ts\" --include=\"*.tsx\"\n\n# Chercher les dangerouslySetInnerHTML\ngrep -r \"dangerouslySetInnerHTML\" apps/web/src --include=\"*.ts\" --include=\"*.tsx\"\n```\n\n### 2. Lancer les vérifications\n\n```bash\n# ESLint\nnpm run lint\n\n# Tests\nnpm run test\n\n# Vérifications globales\nnpm run checks\n```\n\n### 3. Utiliser la checklist\n\nAvant de faire un commit, vérifier :\n- [ ] Pas de `startsWith(\"http\")` pour valider des URLs\n- [ ] Pas de `innerHTML` avec contenu utilisateur\n- [ ] Pas de `dangerouslySetInnerHTML` sans sanitisation\n- [ ] Utiliser `textContent` pour du texte simple\n- [ ] Utiliser `new URL()` pour parser les URLs\n\n---\n\n## 👀 Pendant la revue de code\n\n### 1.
