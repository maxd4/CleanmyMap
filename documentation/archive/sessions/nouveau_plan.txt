Pour éviter les fuites de secrets au push GitHub, les plus importants sont :

.gitignore
.env.example
.env.local non tracké
.env non tracké
.dockerignore si Docker
README.md avec section variables d'environnement
éventuellement un fichier de doc type documentation/securite/secrets.md
Point important

Le .gitignore protège seulement les fichiers non encore suivis.
Si un secret a déjà été commité une fois, l'ajouter ensuite au .gitignore ne suffit pas. Il faut alors :

retirer le fichier du suivi Git
faire tourner les secrets exposés
parfois nettoyer l'historique Git si l'exposition est réelle
