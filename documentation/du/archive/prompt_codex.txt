Usage: prioriser les défauts de perception et d'usage.

Objectif : Supprimer la perception d'attente. Remplacer par anticipation, fluidité et feedback immédiat.

Prompt:
"Fais un audit UX réel du site avec les 10 critères fournis (hiérarchie, densité, micro-interactions, cohérence, navigation, performance perçue, direction artistique, différenciation, contenu, comportement utilisateur).
Écris le rendu dans le fichier texte documentation/du/UX.txt

Contraintes :

- Préserver cohérence UI
- Pas d'animations inutiles
- UX prioritaire

Actions obligatoires :

1. Loading UX
- Supprimer spinners génériques
- Implémenter skeleton screens
- Skeleton = reflet réel du contenu (avatars, lignes texte, blocs)
- Éviter rectangles génériques

2. Structure skeleton
- Avatar → cercle
- Titre → ligne courte
- Texte → lignes longues

3. Motion skeleton
- Ajouter shimmer directionnel gauche → droite

- Actions utilisateur instantanées (like, submit, toggle…)
- Mettre à jour UI immédiatement sans attendre serveur

6. Feedback utilisateur
- Chaque action = réponse visuelle immédiate
- Supprimer toute latence perçue
- Éviter états "figés" ou ambigus

Méthode :
- Identifier tous les états de chargement
- Remplacer spinners par skeletons adaptés
- Implémenter optimistic updates sur actions clés
- Harmoniser animations et timings
- Tester perception utilisateur (rapidité ressentie)

Output attendu :
- Liste des écrans avec mauvais loading UX
- Nouveau système skeleton (règles + exemples)
- Implémentation optimistic UI
- Fichiers modifiés
- Code appliqué
- Vérification UX finale (fluidité + perception vitesse)

Format strict:
1) Résumé exécutif
2) Top problèmes prioritaires
3) Audit détaillé par critère
4) Corrections recommandées par priorité
5) Points nécessitant des données réelles
6) Quick wins avant prochaine démo.

Contraintes: critique, concret, séparer faits/inférences, pas de blabla."

11.5) Audit UI réel (écran par écran)
Usage: prioriser les défauts de perception et d'usage.

Objectif : 
Agis comme un expert design system + UI engineer senior. Audite puis refonds mon site web pour corriger toute incohérence visuelle et le faire reposer sur un vrai design system maintenable, cohérent et scalable.

Prompt:
"Fais un audit UI réel du site et écris le dans le fichier texte documentation/du/UI.txt avec un plan step by step d'implémentation.


Applique strictement ces 5 axes :
1. Couleurs sémantiques : remplace les couleurs hardcodées par des tokens fonctionnels (background, surface, text, border, brand, success, warning, error, muted, info) avec une échelle cohérente de nuances.
2. Typographie : remplace les tailles arbitraires par une hiérarchie structurée (display, h1-h4, body, small, caption, label) avec size, weight et line-height explicites.
3. Spacing : impose un système d'espacement basé sur une unité constante (ex. 4px) pour tous les margin, padding et gaps.
4. Composants : standardise les composants réutilisables et définis tous leurs états (default, hover, active, focus, disabled, loading, error, success).
5. Motion : rends les animations utiles, sobres, cohérentes, non bloquantes, avec durations adaptées et interactions <= 300ms.

Objectif :
- supprimer l'aspect aléatoire
- améliorer lisibilité, cohérence, crédibilité, maintenabilité
- centraliser les règles UI
- préserver performances, accessibilité, responsivité et clarté fonctionnelle

Méthode :
- analyse l'existant
- détecte incohérences et anti-patterns
- définis le système cible
- implémente les tokens et composants
- refactorise le code pour consommer ce système
- évite les changements superficiels
- évite la coexistence d'anciens styles incohérents







Voici la suite de l'audit à réaliser et donc le contenu dout être écirt à la suite du précédent dans le fichier texte documentation/du/UI.txt au meme format step by step.

Usage : Construire ou refactorer un design system complet basé sur tokens.

Objectif :
Remplacer toute UI arbitraire par un système cohérent, modulaire, scalable où chaque pixel est intentionnel.

Contraintes :
- Une seule source de vérité (tokens)
- Cohérence globale stricte
- Maintenabilité maximale
- Pas de styles isolés ou hardcodés

Actions obligatoires :

1. Color system
- Choisir une stratégie claire :
  - monochromatique (1 hue + variations lightness)
  - complémentaire (contraste)
  - analogue (harmonie)
- Construire palette cohérente
- Décliner en tokens sémantiques
- Assigner rôles précis (background, surface, text, brand, states)
- Utilise des dégradés avec des couleurs à seulement 60 degrés sur la roue des couleurs autour de la couleur cible 
- Les dégradés doiven avoir une direction  d 135° 
- Utilise un gradient maillé à 3 points de couleur positionnés aléatoirement
- dégradés sur les texte uniquement pour les titres 
-
2. Typography
- Supprimer tailles arbitraires
- Implémenter modular scale (ratio constant)
- Niveaux : display, body, caption (+ variantes si besoin)
- 1 seule font family
- Définir size / weight / line-height
- Hiérarchie immédiate et cohérente

3. Spacing system
- Base unit = 8px
- Échelle : 8, 16, 24, 32, 40…
- Appliquer à tous margins / paddings / gaps
- Supprimer tout spacing non multiple

4. Core tokens
- Définir tokens fondamentaux :
  - color
  - typography
  - spacing
  - radius
  - shadow (si utilisé)
- Centraliser dans un système unique

5. Component architecture
- Appliquer logique atomic design :
  - atoms → styles de base
  - molecules → composants simples (button, input)
  - organisms → blocs complexes (card, sections)
- Construire composants uniquement via tokens
- Interdire styles locaux incohérents

6. Cohérence globale
- Chaque propriété UI doit dériver d'un token
- Aucun pixel arbitraire
- Uniformiser tout le produit

Méthode :
- Audit complet UI
- Identifier incohérences
- Définir tokens système
- Refactor composants → consommer tokens
- Supprimer duplication et styles legacy

Output attendu step by step dans le fichier texte :
- Diagnostic incohérences
- Système de tokens (color, type, spacing, radius)
- Échelle typographique + spacing
- Architecture composants (atoms/molecules/organisms)
- Vérification cohérence globale

23) SEO full audit 
Usage: structurer une trajectoire SEO complète.

Prompt:
"Audit SEO complet selon 5 piliers: structure, speed, content quality, metadata, backlinks strategy.
Write a step by step plan in documentation/du/SEO.txt

- SEO (Search Engine Optimization) : optimisation technique et sémantique d'un site pour améliorer son classement dans les moteurs de recherche comme Google.
- AEO (Answer Engine Optimization) : optimisation du contenu pour apparaître directement comme réponse dans les moteurs de réponse et assistants vocaux comme ChatGPT ou Google Assistant.
- GEO (Generative Engine Optimization) : optimisation pour être cité ou utilisé par les IA génératives dans leurs réponses, en structurant le contenu pour les modèles comme ChatGPT.
- SXO (Search Experience Optimization) : combinaison SEO + UX visant à améliorer à la fois le classement et l'expérience utilisateur pour maximiser conversion et engagement.
- AIO (AI Optimization) : optimisation globale d'un produit ou site pour l'intégration, l'exploitation et la performance des systèmes d'intelligence artificielle.


Output:
1) Diagnosis by pillar
2) Weak points + root causes
3) Prioritized corrections
4) Concrete implementation plan
Then transform audit into a series of Codex prompts to execute fixes."
