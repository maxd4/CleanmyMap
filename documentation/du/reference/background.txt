Analyse chaque rubrique/page/section du site et choisis le background le plus pertinent pour chacune au lieu d’appliquer un fond global unique. Écrit cet audit au debut de ce fichier texte "background" puis écris une succession de prompt a t'envoyer pour implémenter ton audit proprement.

Options possibles à arbitrer intelligemment :
- couleur unie sobre
- dégradé léger statique
- dégradé doux animé lentement
- mesh gradient discret
- texture topographique légère
- texture cartographique abstraite légère
- trame/grille subtile
- parallax très léger
- fond fixe avec contenu scrollant
- halo diffus discret
- glassmorphism très léger sur certains blocs
- aucun effet si c’est meilleur pour la lisibilité

Évalue notamment :
- accueil / hero
- carte principale
- listes / résultats
- fiche détail
- formulaires / participation
- dashboard / admin
- auth
- pages info
- CTA
- états vides
- modales
- sidebars / filtres / drawers
- footer

Critères obligatoires :
- lisibilité
- hiérarchie visuelle
- cohérence avec un produit civic-tech écologique premium
- présence de carte
- densité d’information
- performance mobile
- accessibilité / contraste
- risque de distraction
- cohérence fonctionnelle

Pour chaque rubrique, donne :
- background recommandé
- intensité
- animation ou non
- justification UX / produit
- risques à éviter
- priorité d’implémentation

Contraintes :
- ne jamais gêner la lecture
- ne jamais concurrencer les schémas, illustration, cartes, fichiers déjà visibles dans la rubrique
- ne jamais faire gadget / flashy / cheap
- privilégier la subtilité
- si un fond simple  est meilleur, garde-le
- limiter la palette finale pour garder une cohérence globale
