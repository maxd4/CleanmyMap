**Bruits / instabilités identifiés**
- `build` bruité par warnings Sentry (non bloquant).
- `run_checks.ps1` peut salir le workspace (auto-fix encodage) si `-SkipEncodingAutofix` absent.

- `npm test` racine n’est pas un vrai test gate (placeholder).

**Protocole minimal incrémental recommandé (court, stable, reproductible)**
1. **À chaque commit local (rapide, ~1-3 min)**  
   - `python scripts/check_runtime_db_tracking.py --root .`  
   - `python scripts/ci_cleanup.py --root . --check`  
   - `npm.cmd --prefix apps/web run lint`  
   - `npm.cmd --prefix apps/web run test`
2. **Avant push / merge (moyen, ~2-6 min)**  
   - `npm.cmd --prefix apps/web run build`  
   - `powershell -ExecutionPolicy Bypass -File scripts/run_checks.ps1 -Scope changed -SkipE2E -SkipEncodingAutofix`
3. **Gate pré-release uniquement**  
   - `powershell -ExecutionPolicy Bypass -File scripts/run_checks.ps1 -Scope full`  
   - puis `npm.cmd run test:e2e` si release UI critique

Ce set est le plus court actuellement sans E2E lourde et avec un vrai filet anti-régression.
